RAM (EMC) overclocking delivers some of the most impactful system-wide performance gains in Horizon OC — higher memory bandwidth directly benefits both CPU and GPU workloads, often more noticeably than raising either compute module's clock alone. However, it is also the riskiest change you can make: the Tegra EMC controller is tightly coupled to NAND flash I/O paths in Horizon OS, and an unstable RAM frequency can cause filesystem corruption on both the SD card and internal NAND storage. Always create a full backup before changing EMC clocks.

<Warning>
**RAM overclocking can cause NAND or SD card corruption.** Before changing any EMC clock setting, ensure you have a complete backup of your NAND, PROINFO, EMUMMC, and SD card contents. This is not optional. A failed RAM overclock can render your console unable to boot without a NAND restore.
</Warning>

## Platform Limits

| Console | SoC | LPDDR4X Max (Horizon OC) | MT/s |
|---------|-----|--------------------------|------|
| Erista (V1 Icosa) | T210 | 2400 MHz | 4800 MT/s |
| Mariko (V2 Iowa, Lite Hoag, OLED Aula) | T210B01 / T214 | 3200 MHz | 6400 MT/s |

The Erista EMC controller tops out at the LPDDR4X JEDEC 4800 standard (2400 MHz). Mariko's newer EMC controller supports up to JEDEC 6400 (3200 MHz) with appropriate timing configuration.

## Available Timing Folders

Horizon OC ships pre-tuned timing sets for the four standard JEDEC breakpoints on Mariko (`timings/Mariko/AA/`):

- `1331` — Official handheld baseline timings
- `1600` — Official docked baseline timings
- `1866` — JEDEC 3733 module timings
- `2133` — JEDEC 4266 module timings

Timings for non-JEDEC frequencies above 2133 MHz are interpolated or carried from the 2133 MHz set. Individual timing parameters (`t1_trcd`, `t2_trp`, `t3_tras`, etc.) can be fine-tuned manually via the KIP config for advanced users.

## Key Config Values

The following config keys control EMC frequency and voltage:

| Config Key | Description |
|---|---|
| `mariko_emc_max_clock` | Maximum EMC clock for Mariko, in kHz |
| `erista_emc_max_clock2` | Maximum EMC clock for Erista, in kHz |
| `common_emc_mem_volt` | VDD2 voltage for the EMC/DRAM rails (both platforms) |
| `mariko_emc_vddq_volt` | VDDQ voltage for Mariko DRAM (Mariko only) |

<Tip>
Start at a JEDEC-standard frequency for your RAM module rating (1866 MHz or 2133 MHz for most Mariko units) before pushing higher. JEDEC speeds use well-validated timing parameters and are significantly more stable than non-standard intermediate steps. Only move to frequencies above your module's JEDEC max after confirming stability, and increase in small increments — 33–66 MHz steps — with thorough testing between each change.
</Tip>

## Voltage Interaction

Higher EMC clocks typically require raising `common_emc_mem_volt` (VDD2) and, on Mariko, `mariko_emc_vddq_volt` (VDDQ). The SOC voltage also scales with RAM frequency on Mariko via the `g_soc_volt_*` per-frequency array. See [Voltage Tuning](/clocks/voltage-tuning) for a full explanation of each rail and safe tuning ranges.
