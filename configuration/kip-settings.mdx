KIP settings are a separate layer of configuration from clock profiles. While profiles control which frequencies `hoc-clk` requests at runtime, KIP settings are parameters baked directly into the `hoc.kip` kernel patch at boot time. They control voltage tables, overclock ceilings, memory timings, and other low-level values that the operating system reads before the `hoc-clk` sysmodule ever starts. Changes to KIP settings do not take effect until you explicitly save them and reboot the console.

## What KIP Settings Are

The `hoc.kip` patch embeds a `CustomizeTable` struct (defined in `customize.hpp`) that Atmosphere's loader applies during kernel initialization. This table carries a revision tag (`CUST_REV = 4`) and a version identifier (`KIP_VERSION = 241`) that Horizon OC uses to verify compatibility. All voltage tables, EMC clock limits, CPU/GPU overclock ceilings, and memory timing overrides live inside this struct.

Because these values are consumed at boot time by the patched loader — before any userland process runs — they cannot be changed live. You must:

1. Adjust the values through the overlay's Settings submenus.
2. Press **Save KIP Settings**.
3. Reboot the console.

<Warning>
KIP settings only take effect after saving **and** performing a full cold reboot. A sleep/wake cycle is not sufficient. Failing to reboot after saving means the currently running system still uses the previous KIP table.
</Warning>

## Accessing KIP Settings

All KIP-modifiable parameters are grouped into four submenus under **Settings** in the overlay:

| Submenu | Contains |
|---|---|
| **RAM Settings** | EMC clocks, EMC voltages, step mode, memory timings |
| **CPU Settings** | Per-platform CPU max clock, boost clock, undervolt, voltage limits |
| **GPU Settings** | Per-platform GPU undervolt level, per-frequency voltage table |
| **Display Settings** | Display refresh rate and max handheld display clock (non-KIP, runtime) |

To save all pending KIP changes, press **Save KIP Settings** at the top of the main Settings menu.

## CUST Revision and KIP Version

```c
#define CUST_REV    4
#define KIP_VERSION 241
```

`CUST_REV` identifies the layout version of the `CustomizeTable` struct. `KIP_VERSION` identifies the specific `hoc.kip` binary. The overlay reads these values from the running `hoc-clk` context and will warn you if there is a mismatch between the installed KIP and the sysmodule.

## RAM / EMC Settings

These keys control the EMC (External Memory Controller) and LPDDR4 voltage/clock configuration embedded in the KIP.

<ParamField path="common_emc_mem_volt" type="u32">
  **Common EMC/MEM Voltage.** VDD2 rail voltage applied to LPDDR4 memory. Shared across Erista and Mariko. Higher values may be required at higher EMC clocks; lower values reduce heat at stock clocks.
</ParamField>

<ParamField path="mariko_emc_max_clock" type="u32">
  **Mariko EMC Max Clock.** Maximum EMC frequency (in kHz) allowed on Mariko (T214/T210B01) hardware. Values above the JEDEC-rated maximum require adequate `common_emc_mem_volt` and a compatible DRAM IC.
</ParamField>

<ParamField path="erista_emc_max_clock2" type="u32">
  **Erista EMC Max Clock.** Maximum EMC frequency (in kHz) allowed on Erista (T210) hardware (Icosa/Copper).
</ParamField>

<ParamField path="mariko_emc_vddq_volt" type="u32">
  **Mariko EMC VDDQ Voltage.** VDDQ rail voltage for LPDDR4 on Mariko. This rail supplies the I/O termination voltage. Typically kept at stock (600 mV) unless pushing very high EMC clocks.
</ParamField>

<ParamField path="soc_vmax" type="u32">
  **SOC Vmax.** Maximum voltage (mV) allowed on the VDD_SOC rail. Raising this enables higher EMC and GPU frequencies on Mariko but increases heat output and battery draw.
</ParamField>

<ParamField path="step_mode" type="StepMode (u32)">
  **Step Mode.** Controls the EMC frequency step granularity used when the sysmodule changes RAM clocks. Corresponds to the `StepMode` enum:

  | Value | Enum | Description |
  |---|---|---|
  | `0` | `StepMode_66MHz` | 66 MHz steps |
  | `1` | `StepMode_100MHz` | 100 MHz steps |
  | `2` | `StepMode_Jedec` | JEDEC-defined steps |
  | `3` | `StepMode_133MHz` | 133 MHz steps |
</ParamField>

<Tip>
Start EMC overclocking with `StepMode_Jedec` (`2`) and `common_emc_mem_volt` raised by 25–50 mV above stock before attempting frequencies above 2133 MHz. Stability issues at high clocks are most often voltage-related rather than frequency-ceiling-related.
</Tip>

## CPU Settings

CPU KIP parameters are split by SoC generation. Mariko (T214/T210B01) and Erista (T210) have distinct voltage tables and different sets of adjustable parameters.

### Mariko CPU (T214 / T210B01)

<ParamField path="mariko_cpu_uv_low" type="u32">
  **Mariko CPU Undervolt (Low).** Undervolt amount (mV) applied to the lower range of the CPU voltage table. Reduces heat and power draw on the Cortex-A57 cluster at moderate frequencies.
</ParamField>

<ParamField path="mariko_cpu_uv_high" type="u32">
  **Mariko CPU Undervolt (High).** Undervolt amount (mV) applied to the upper range of the CPU voltage table. Used when the CPU is running at or near its maximum clock.
</ParamField>

<ParamField path="mariko_cpu_max_clock" type="u32">
  **Mariko CPU Max Clock.** Absolute maximum CPU frequency (in kHz) the platform will allow on Mariko hardware. The overlay `cpu_max_m` config value (`MarikoMaxCpuClock`) references this ceiling. Default display value: 1963 MHz.
</ParamField>

<ParamField path="mariko_cpu_boost_clock" type="u32">
  **Mariko CPU Boost Clock.** The frequency (in kHz) the CPU targets during boost mode events triggered by the system. Overwriting boost mode via `ow_boost` replaces this with the profile frequency.
</ParamField>

<ParamField path="mariko_cpu_max_volt" type="u32">
  **Mariko CPU Max Voltage.** Hard ceiling on CPU rail voltage (mV). Prevents the DVFS table from requesting voltages above this value even if a frequency entry calls for more.
</ParamField>

### Erista CPU (T210)

<ParamField path="erista_cpu_uv" type="u32">
  **Erista CPU Undervolt.** Undervolt amount (mV) subtracted uniformly from all CPU DVFS voltage entries on Erista hardware.
</ParamField>

<ParamField path="erista_cpu_max_volt" type="u32">
  **Erista CPU Max Voltage.** Hard ceiling on CPU rail voltage (mV) for Erista.
</ParamField>

<ParamField path="erista_cpu_boost_clock" type="u32">
  **Erista CPU Boost Clock.** Boost mode target frequency (in kHz) for Erista hardware.
</ParamField>

<Tip>
A conservative starting point for CPU undervolting is 6250 µV (6 mV) for `mariko_cpu_uv_low` / `mariko_cpu_uv_high` on Mariko, or `erista_cpu_uv` on Erista. Increase in small increments and verify stability with a CPU-intensive workload before saving.
</Tip>

## GPU Settings

GPU KIP parameters control the per-frequency voltage table used by the Maxwell (Erista) and Ampere-class (Mariko) GPUs. Each platform has its own array of per-step voltages in addition to scalar undervolt and ceiling parameters.

### Mariko GPU (T214 / T210B01)

<ParamField path="mariko_gpu_uv" type="u32">
  **Mariko GPU Undervolt.** Selects the GPU undervolt level preset. Corresponds to the `GPUUndervoltLevel` enum (`NoUV=0`, `SLT=1`, `HiOPT=2`, `HiOPT15=3`, `HighUV=4`). Presets scale the entire Mariko GPU voltage table.
</ParamField>

<ParamField path="mariko_gpu_vmax" type="u32">
  **Mariko GPU Vmax.** Maximum voltage (mV) the GPU rail is allowed to reach on Mariko.
</ParamField>

<ParamField path="mariko_gpu_boot_volt" type="u32">
  **Mariko GPU Boot Voltage.** Voltage (mV) applied to the GPU rail at boot, before the DVFS governor takes over.
</ParamField>

### Erista GPU (T210)

<ParamField path="erista_gpu_uv" type="u32">
  **Erista GPU Undervolt.** Selects the GPU undervolt level preset for Erista hardware.
</ParamField>

<ParamField path="erista_gpu_vmin" type="u32">
  **Erista GPU Minimum Voltage.** Floor voltage (mV) below which the GPU rail will not be set, even at idle.
</ParamField>

### Per-Frequency Voltage Tables

In addition to the scalar parameters above, the KIP table stores a complete per-frequency voltage array for each platform. These give you fine-grained control over exactly how much voltage each GPU step requests.

<Warning>
Running GPU frequencies above **1075 MHz without an undervolt preset** (or above **1152 MHz on the SLT table** / **1228 MHz on HiOPT**) on Mariko can cause permanent damage. On Erista, frequencies above **921 MHz without UV** or **960 MHz on SLT/HiOPT** carry the same risk. Always use the appropriate undervolt preset before raising GPU clocks above these thresholds.
</Warning>

## Save and Apply Workflow

<Steps>
  <Step title="Adjust KIP values">
    Navigate to **Settings → RAM Settings**, **CPU Settings**, or **GPU Settings** and modify the desired parameters using the overlay controls.
  </Step>
  <Step title="Press Save KIP Settings">
    Return to the main Settings menu and press **A** on **Save KIP Settings**. The overlay sends the updated `CustomizeTable` to the `hoc-clk` sysmodule, which writes it into the active KIP patch on the SD card.
  </Step>
  <Step title="Reboot the console">
    Perform a full cold reboot. Atmosphere's loader reads the updated KIP table during kernel initialization. The new voltages, clocks, and timing values are now active.
  </Step>
  <Step title="Verify stability">
    Run a workload that exercises the overclocked component (a demanding game for GPU/CPU, a RAM-intensive title for EMC) and monitor temperatures via the overlay's header display. Roll back any value that causes crashes or artifacts.
  </Step>
</Steps>

<Tip>
If the system fails to boot after a KIP settings change, hold **Power** for 12 seconds to force-power-off, then boot while holding **Vol−** to enter Atmosphere's recovery menu (Hekate or fusee). From there you can restore a backup of your `hoc.kip` or reset KIP settings to defaults.
</Tip>
